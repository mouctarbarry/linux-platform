## Syntaxe

```bash
ping [OPTIONS] HÔTE
```

## Options principales

| Option | Description                              |
| ------ | ---------------------------------------- |
| `-c`   | Nombre de paquets à envoyer              |
| `-i`   | Intervalle entre les paquets en secondes |
| `-W`   | Délai d'attente de réponse en secondes   |
| `-s`   | Taille du paquet en octets               |

$ ping -c 4 google.com

64 bytes from par21s20-in-f14.1e100.net: icmp_seq=4 ttl=118 time=4.05 ms

### Tester la connectivité avec un délai court

```bash
$ ping -c 2 -W 1 192.168.1.1
PING 192.168.1.1 (192.168.1.1) 56(84) bytes of data.
64 bytes from 192.168.1.1: icmp_seq=1 ttl=64 time=0.842 ms
64 bytes from 192.168.1.1: icmp_seq=2 ttl=64 time=0.731 ms

--- 192.168.1.1 ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1001ms
rtt min/avg/max/mdev = 0.731/0.786/0.842/0.055 ms
```

### Détecter un hôte injoignable

```bash
$ ping -c 3 -W 2 10.0.0.99
PING 10.0.0.99 (10.0.0.99) 56(84) bytes of data.

--- 10.0.0.99 ping statistics ---
3 packets transmitted, 0 received, 100% packet loss, time 4028ms
```
